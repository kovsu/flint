Flint's CLI comes as part of the [`flint` npm package](https://npmjs.com/package/flint).
You can run it with `npx` or an equivalent:

<PackageManagers pkg="flint" type="dlx" />

## `--cache-ignore`

Whether to ignore any existing cache data on disk.
This will cause a full re-lint of all linted files.

<PackageManagers args="--cache-ignore" pkg="flint" type="dlx" />

:::tip
Flint stores linting caches under `node_modules/.cache/flint.json`.
Open that file in an editor to see which files are cached as well as dependencies between files.
:::

## `--fix`

Enables auto-fixing ["fixes"](/glossary#fix) from rule reports.

<PackageManagers args="--fix" pkg="flint" type="dlx" />

Rule reports may include safe "fixer" changes that show how to fix source code.
Adding `--fix` will instruct Flint to continuously re-run (up to 10 times), fixing code and re-linting files.

## `--fix-suggestions`

Enables auto-fixing any number of specific ["suggestions"](/glossary#suggestion) from rule reports.

<PackageManagers
	args="--fix-suggestions cspell:addWordToWords"
	pkg="flint"
	type="dlx"
/>

Unlike [`--fix`](#--fix), `--fix-suggestions` requires passing in explicit names of suggestions to apply.
Suggestion names are comprised of the rule name, and a suggestion ID, all joined by `:`.

## `--skip-diagnostics`

Whether to skip reporting language "diagnostics" after linting.

<PackageManagers args="--skip-diagnostics" pkg="flint" type="dlx" />

Language plugins such as Flint's TypeScript plugin are able to report additional checks from the languages after linting.
For example, Flint's core TypeScript plugin reports TypeScript type errors (effectively running `tsc --noEmit`) by re-using the TypeScript type services used in linting.

`--skip-diagnostics` can be used to disable those checks.
You may wish to do this for performance and/or if you run tools like `tsc` separately from Flint.

## `--version`

Prints the current package version of Flint.

<PackageManagers args="--version" pkg="flint" type="dlx" />

## `--watch`

Whether to keep the linting process running, re-linting files as they change.

<PackageManagers args="--watch" pkg="flint" type="dlx" />

Flint's `--watch` mode is similar to the watch mode in other developer tools CLIs such as [`tsc --watch`](https://www.typescriptlang.org/docs/handbook/compiler-options.html#:~:text=Watch%20Options,-Flag), [`tsdown --watch`](https://tsdown.dev/options/watch-mode), and [`vitest --watch`](https://vitest.dev/guide/cli.html#watch).
It will lint files once on startup, then again whenever any linted files are changed on disk.
