The `document.domain` API changes the origin the browser uses for same-origin checks.
Setting it can let pages on related subdomains access each other when the browser would otherwise keep them isolated.
Use `location.hostname` or `location.origin` when you only need to read the current page's host or origin.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```ts
document.domain = "example.com";
```

```ts
window.document.domain = "example.com";
```

</TabItem>
<TabItem label="✅ Correct">

```ts
const currentHost = location.hostname;
```

```ts
const currentOrigin = location.origin;
```

</TabItem>
</Tabs>

## Options

This rule is not configurable.

## When Not To Use It

If you maintain a legacy app that temporarily depends on cross-subdomain same-origin relaxation while migration is in progress, it may be reasonable to disable this rule for those files.
For new code, prefer current host checks or server-side origin configuration instead.
