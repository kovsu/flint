## Presets

Flint's package.json plugin provides the following preset:

| Preset                        | Recommended   | Description                                                             |
| ----------------------------- | ------------- | ----------------------------------------------------------------------- |
| **[`logical`](#logical)**     | ✅ Always     | Common rules for finding bugs and good practices in package.json files. |
| **[`sorting`](#sorting)**     | ☑️ When Ready | Common rules sorting various aspects of code.                           |
| **[`stylistic`](#stylistic)** | ✅ Always     | Common rules for consistent styling and best stylistic practices.       |

Flint recommends using the [`logical`](#logical) and [`stylistic`](#stylistic) presets:

```ts title="flint.config.ts"
import { defineConfig, packageJson } from "flint";

export default defineConfig({
	use: [
		{
			files: packageJson.files.all,
			rules: [packageJson.presets.logical, packageJson.presets.stylistic],
		},
	],
});
```

### `logical`

Rules that find bugs and enforce best practices and prevent common pitfalls for most-to-all package.json files.

```ts title="flint.config.ts"
import { defineConfig, packageJson } from "flint";

export default defineConfig({
	use: [
		{
			files: packageJson.files.all,
			rules: packageJson.presets.logical,
		},
	],
});
```

### `sorting`

Rules that enforce consistent ordering of various aspects of code.

```ts title="flint.config.ts"
import { defineConfig, packageJson } from "flint";

export default defineConfig({
	use: [
		{
			files: packageJson.files.all,
			rules: packageJson.presets.sorting,
		},
	],
});
```

### `stylistic`

Rules that enforce consistent styling and best stylistic practices for most-to-all package.json files.

```ts title="flint.config.ts"
import { defineConfig, packageJson } from "flint";

export default defineConfig({
	use: [
		{
			files: packageJson.files.all,
			rules: packageJson.presets.stylistic,
		},
	],
});
```
