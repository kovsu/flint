npm requires that the `author` field in `package.json` follows certain specifications if it is present.
The field can be either a string or an object with specific properties.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"author": ""
}
```

```json
{
	"author": null
}
```

```json
{
	"author": "Name <invalid>"
}
```

```json
{
	"author": {}
}
```

```json
{
	"author": {
		"name": "Name",
		"email": "invalid"
	}
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"author": "First Last"
}
```

```json
{
	"author": "Name <name@example.com>"
}
```

```json
{
	"author": {
		"name": "Name"
	}
}
```

```json
{
	"author": {
		"name": "Name",
		"email": "name@example.com",
		"url": "https://example.com"
	}
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `author` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
