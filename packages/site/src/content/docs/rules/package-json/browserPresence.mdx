<RuleSummary plugin="package-json" rule="browserPresence" />

This rule requires the `browser` field in `package.json` to be present.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{}
```

```json
{
	"name": "my-repository"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"browser": "./browser.js"
}
```

```json
{
	"browser": "./browser.js",
	"name": "my-repository"
}
```

</TabItem>
</Tabs>

## Options

### `ignorePrivate`

Whether the property should still be required when the package's `private` property is `true`.
Defaults to `false`.

This option can be useful if the rule is enabled on many `package.json` files and some files differ in privacy.

## When Not To Use It

If your package does not need a browser-specific entry point, you can disable this rule.
For example, a package imported with modern `exports`, or that only runs in Node.js, may not need a `browser` field.
