The rule checks that, if present, the `config` property is an object.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"config": true
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"config": {
		"port": "8080"
	}
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `config` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
