<RuleSummary plugin="package-json" rule="contributorsPresence" />

npm requires that the `contributors` field in `package.json` is present.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{}
```

```json
{
	"name": "my-repository"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"contributors": [{ "name": "First Last" }]
}
```

```json
{
	"contributors": [{ "name": "First Last" }],
	"name": "my-repository"
}
```

</TabItem>
</Tabs>

## Options

### `ignorePrivate`

Whether the property should still be required when the package's `private` property is `true`.
Defaults to `false`.

This option can be useful if the rule is enabled on many `package.json` files and some files differ in privacy.

## When Not To Use It

If your package doesn't need to specify `contributors`, you can disable this rule.
For example, if your package is local-only and is never published online or made open source, contributor attribution may not be relevant for you.
