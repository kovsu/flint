The rule checks that, if present, the `cpu` property is an array of non-empty strings.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"cpu": true
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"cpu": ["x64", "ia32"]
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `cpu` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
