<RuleSummary plugin="package-json" rule="descriptionValidity" />

This rule ensures that the `description` field is a non-empty string.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"description": ""
}
```

```json
{
	"description": null
}
```

```json
{
	"description": {}
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"description": "The Fragile"
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `description` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
