The rule checks that, if present, the `directories` property is an object.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"directories": true
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"directories": {
		"bin": "dist/bin",
		"man": "docs"
	}
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `directories` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
