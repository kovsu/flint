This rule checks that the `engines` property is an object with non-empty string values.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"engines": {
		"node": 24
	}
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"engines": {
		"node": "^24.11.0"
	}
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `engines` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
