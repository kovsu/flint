This rule reports entries in `files` that npm includes automatically.
npm always includes `package.json`, root-level README, COPYING, LICENSE, and LICENCE files, the file declared in `main`, and files declared in `bin`.
It also reports duplicate `files` entries.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"files": ["README.md", "dist/"]
}
```

```json
{
	"main": "./lib/index.js",
	"files": ["lib/index.js", "dist/"]
}
```

```json
{
	"files": ["dist/", "dist/"]
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"files": ["CHANGELOG.md", "dist/"]
}
```

```json
{
	"main": "./lib/index.js",
	"files": ["dist/"]
}
```

</TabItem>
</Tabs>

## When Not To Use It

If a package uses a custom publishing tool that does not follow npm's automatic package-file inclusion rules, this rule might not apply.
For example, another tool might require README or entry-point files to be listed explicitly.
