This rule checks that the `files` property is an array of non-empty strings.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"files": "dist/*"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"files": ["CHANGELOG.md", "dist/"]
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `files` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
