This rule checks that the `gypfile` property is a boolean.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"gypfile": "true"
}
```

```json
{
	"gypfile": {}
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"gypfile": true
}
```

```json
{
	"gypfile": false
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `gypfile` field specification, you can disable this rule.
For example, if your package metadata is consumed by tooling that accepts non-boolean `gypfile` values, this rule may not be correct for you.
