npm requires that the `main` field in `package.json` is present.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{}
```

```json
{
	"name": "my-repository"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"main": "./lib/index.js"
}
```

```json
{
	"main": "./lib/index.js",
	"name": "my-repository"
}
```

</TabItem>
</Tabs>

## Options

### `ignorePrivate`

Whether the property should still be required when the package's `private` property is `true`.
Defaults to `false`.

This option can be useful if the rule is enabled on many `package.json` files and some files differ in privacy.

## When Not To Use It

If your package doesn't need to specify a name, you can disable this rule.
For example, if your package is local-only and is never published online or made open source, naming may not be relevant for you.
