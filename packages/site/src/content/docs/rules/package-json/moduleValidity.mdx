This rule checks that the `module` property is a non-empty string.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"module": ["index.js", "secondary.js"]
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"module": "index.js"
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `module` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
