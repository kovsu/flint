<RuleSummary plugin="package-json" rule="optionalDependenciesPresence" />

npm requires that the `optionalDependencies` field in `package.json` is present.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{}
```

```json
{
	"name": "my-repository"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"optionalDependencies": {}
}
```

```json
{
	"name": "my-repository",
	"optionalDependencies": {}
}
```

</TabItem>
</Tabs>

## Options

### `ignorePrivate`

Whether the property should still be required when the package's `private` property is `true`.
Defaults to `false`.

This option can be useful if the rule is enabled on many `package.json` files and some files differ in privacy.

## When Not To Use It

If your package doesn't need to specify `optionalDependencies`, you can disable this rule.
