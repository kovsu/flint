<RuleSummary plugin="package-json" rule="peerDependenciesMetaPresence" />

This rule requires the `peerDependenciesMeta` field in `package.json` to be present.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{}
```

```json
{
	"name": "my-repository"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"peerDependenciesMeta": {
		"example": {
			"optional": true
		}
	}
}
```

```json
{
	"name": "my-repository",
	"peerDependenciesMeta": {
		"example": {
			"optional": true
		}
	}
}
```

</TabItem>
</Tabs>

## Options

### `ignorePrivate`

Whether the property should still be required when the package's `private` property is `true`.
Defaults to `false`.

This option can be useful if the rule is enabled on many `package.json` files and some files differ in privacy.

## When Not To Use It

If your package does not need peer dependency metadata, you can disable this rule.
For example, a package without optional peer dependencies may not need `peerDependenciesMeta`.
