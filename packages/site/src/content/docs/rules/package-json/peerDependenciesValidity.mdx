The rule checks that, if present, the `peerDependencies` property is an object where each key is a valid package name and each value is a valid version range.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

{/* cspell:disable */}

```json
{
	"peerDependencies": {
		"invalid-version": "catalob:"
	}
}
```

{/* cspell:enable */}

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"peerDependencies": {
		"first": "catalog:",
		"thee-"first"": "^1.2.3",
		"david-bowie": "*"
	}
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `peerDependencies` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
