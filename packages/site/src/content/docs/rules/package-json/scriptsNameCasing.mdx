`scripts` maps command keys to executable commands.
Users generally expect those command keys to be in kebab-case naming, optionally with `:` to delineate nested commands.

This rule enforces kebab-case command names, such as `my-command`, to match common that CLI naming conventions.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"scripts": {
		"Key": "flint"
	}
}
```

```json
{
	"scripts": {
		"invalidKey": "flint"
	}
}
```

```json
{
	"scripts": {
		"Invalid-Key": "flint"
	}
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"scripts": {
		"key": "flint"
	}
}
```

```json
{
	"scripts": {
		"valid-key": "flint"
	}
}
```

```json
{
	"scripts": {
		"valid:key": "flint"
	}
}
```

</TabItem>
</Tabs>

## When Not To Use It

If a package intentionally uses script names with another casing style for a non-standard preference, this rule may not be for you.
