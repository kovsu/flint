This rule checks that the `sideEffects` property is either a boolean or an array.
If the value is an array, it should only consist of non-empty strings.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"sideEffects": "false"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"sideEffects": false
}
```

```json
{
	"sideEffects": ["./dist/polyfills.js"]
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `sideEffects` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
