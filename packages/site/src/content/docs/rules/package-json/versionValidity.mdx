The rule checks that the `version` property is a semver version string.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
	"version": 1
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
	"version": "1.0.0"
}
```

</TabItem>
</Tabs>

## When Not To Use It

If you don't need compliance with npm's `version` field specification, you can disable this rule.
For example, if you use a different package manager that treats the field differently, this rule may not be correct for you.
